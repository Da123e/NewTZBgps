# 黄淮学院学生位置管理系统 - 部署文档和使用说明

## 1. 系统概述

黄淮学院学生位置管理系统是一套基于客户端-服务器架构的学生位置监控系统，主要功能包括：

- 学生端：位置定位、校内/校外判断、定时位置上传、速度检测
- 教师端：学生实时位置监控、历史位置查询、多条件筛选、超速报警
- 服务器端：数据接收、存储、查询接口、超速检测

#### 4.1.2 定位容错机制

系统已实现定位容错机制，解决GPS定位误差问题：
- **容错半径**: 100米（可调整 `LOCATION_TOLERANCE` 参数）
- **判断逻辑**: 
  1. 首先检查点是否在多边形内部
  2. 如果不在内部，计算点到多边形边界的距离
  3. 如果距离在容错范围内（100米），认为在校内
- **定位精度**: 结合设备上报的定位精度信息，取较大值作为实际容错范围

#### 4.2.1 网络连接方式

系统支持以下几种网络连接方式：

**方式1：局域网连接（默认配置）**
- 服务器IP：10.67.247.40（局域网IP）
- 适用场景：手机和电脑连接同一个WiFi/路由器
- 优点：配置简单，无需额外工具
- 缺点：受网络限制，必须连接同一网络

**方式2：内网穿透（推荐用于测试）**
- 使用ngrok、frp等工具
- 适用场景：开发测试，需要随时随地访问
- 优点：无需购买服务器，快速测试
- 缺点：免费版不稳定，地址会变化

**方式3：云服务器部署（推荐用于正式使用）**
- 部署到阿里云、腾讯云等
- 适用场景：正式使用，需要稳定访问
- 优点：稳定可靠，随时随地访问
- 缺点：需要购买服务器

### 4.3 内网穿透工具使用

#### 4.3.1 使用ngrok（推荐）

ngrok是最简单易用的内网穿透工具：

**安装步骤**：
1. 访问 https://ngrok.com/ 注册账号
2. 下载ngrok客户端（Windows/Mac/Linux）
3. 解压到任意目录

**使用步骤**：
```bash
# 在服务器端目录运行
cd c:\Users\孟壮壮\Desktop\2026-01\服务器端\backend

# 启动ngrok（假设服务器运行在8000端口）
ngrok http 8000
```

**获取公网地址**：
运行后会显示类似信息：
```
Forwarding  http://abc123.ngrok.io -> http://localhost:8000
Forwarding  https://abc123.ngrok.io -> http://localhost:8000
```

**修改客户端配置**：
```javascript
// 使用ngrok提供的HTTP地址
SERVER_URL: 'http://abc123.ngrok.io'
// 或使用HTTPS（推荐）
SERVER_URL: 'https://abc123.ngrok.io'
```

**注意事项**：
- 每次重启ngrok，地址会变化
- 免费版有连接数限制（每个月4g）
- 建议使用HTTPS地址，更稳定

#### 4.3.2 使用frp（稳定方案）

frp是更稳定的内网穿透工具：

**准备工作**：
1. 需要一台有公网IP的服务器（VPS）
2. 下载frp客户端和服务器端

**配置步骤**：

**服务器端配置（frps.ini）**：
```ini
[common]
bind_port = 7000

[ssh]
listen_port = 6000
auth_token = your-token
```

**客户端配置（frpc.ini）**：
```ini
[common]
server_addr = your-vps-ip
server_port = 7000
auth_token = your-token

[web]
type = http
local_ip = 127.0.0.1
local_port = 8000
custom_domains = your-domain.com
```

**使用步骤**：
```bash
# 服务器端运行
./frps -c frps.ini

# 客户端运行（在您的电脑上）
./frpc -c frpc.ini
```

**修改客户端配置**：
```javascript
SERVER_URL: 'http://your-domain.com'
```


```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

**启用配置**：
```bash
# 创建软链接
ln -s /etc/nginx/sites-available/location-tracker /etc/nginx/sites-enabled/

# 测试配置

systemctl restart nginx
```

## 5. 使用说明

### 5.1 用户身份选择

1. 访问系统首页，进入身份选择界面
2. 选择"学生"或"教师"身份

### 5.2 学生端使用

#### 注册
1. 点击"注册"按钮
2. 填写学号、密码、姓名、年级、院(系)/部、专业、班级信息
3. 点击"注册"按钮完成注册

#### 登录
1. 填写学号和密码
2. 点击"登录"按钮

#### 位置管理
1. 登录后进入主界面
2. 点击"开始定位"按钮
3. 系统开始获取位置信息并显示校内/校外状态
4. 系统根据校内/校外状态自动定时上传位置信息：
   - 校内：每3分钟上传一次
   - 校外：每1分钟上传一次
5. 系统使用黄淮学院北区多边形边界判断校内/校外状态（带100米容错）
6. 点击"停止定位"按钮可停止定位和上传

#### 速度检测
- 系统会自动检测学生的移动速度
- 速度信息会随位置数据一起上传
- 速度单位：米/秒（m/s）
- 使用高精度定位模式，确保速度数据准确

### 5.3 教师端使用

#### 注册
1. 点击"注册"按钮
2. 填写工号、密码、姓名、院(系)/部、所授专业信息
3. 点击"注册"按钮完成注册

#### 登录
1. 填写工号和密码
2. 点击"登录"按钮

#### 学生监控
1. 登录后进入学生管理界面
2. 系统自动显示所教专业学生的实时位置和校内/校外状态
3. 在学生信息卡片中实时显示学生速度
4. 点击右上角"刷新"按钮可手动刷新学生状态

#### 多条件筛选
1. 在筛选区域填写年级、院(系)/部、专业或学号
2. 点击"筛选"按钮进行筛选

#### 视图切换
1. 点击"地图视图"或"列表视图"按钮切换显示模式
2. **地图视图**：在地图上显示学生位置，点击标记可查看详细信息
3. **列表视图**：以列表形式显示学生信息，包括状态、速度、更新时间

#### 超速报警
1. 当学生在校外且速度超过80码（35.76米/秒）时，系统会记录超速报警
2. 教师监控页面的"报警"按钮会闪烁提醒
3. 点击"报警"按钮进入报警页面
4. 报警页面显示所有超速学生的详细信息：
   - 学生姓名、学号、班级
   - 超速速度（米/秒和码）
   - 超速位置和时间
   - 报警级别（警告/严重）
5. 报警记录会在30天后自动销毁
6. 查看报警后，报警按钮停止闪烁，直到有新的超速记录

#### 历史位置查询
1. 点击学生列表中的任意学生项
2. 系统弹出历史位置记录窗口
3. 显示该学生最近100次的位置记录
4. **时间筛选**：在窗口顶部的时间筛选器中选择时间范围（全部、最近1小时、最近3小时、最近6小时、最近12小时、最近1天、最近3天、最近7天）
5. **视图切换**：点击"列表"或"地图"按钮切换显示模式
6. **列表视图**：
   - 显示每次定位的时间、坐标、地址
   - 显示校内/校外状态
   - 显示速度信息（超速显示红色，正常显示绿色）
7. **地图视图**：
   - 显示学生位置轨迹，使用渐变色（从浅蓝到深蓝）表示移动方向
   - 支持地图放大和缩小操作
   - 显示起点（绿色）和终点（红色）标记
8. 点击"关闭"按钮关闭窗口

### 6.1 服务器端优化

3. **路由修复**：移除重复的路由定义，确保API接口正常工作
4. **错误处理增强**：添加详细的错误日志记录，便于问题排查
5. **速度检测**：添加超速检测功能，自动识别校外学生超速行为
6. **定位容错**：添加100米容错机制，解决GPS定位误差导致的误判问题
7. **报警记录管理**：实现30天自动清理机制，保持数据时效性
8. **运动权限支持**：添加运动权限配置，确保能够正确获取速度信息

### 6.2 客户端优化

1. **本地边界判断**：实现客户端本地校园边界判断，即使在网络不稳定情况下也能显示正确的校内/校外状态
2. **服务器地址统一**：所有页面使用统一的SERVER_URL配置，确保与服务器通信正常
3. **错误处理优化**：添加详细的错误提示，提高用户体验
4. **地图功能**：实现学生位置轨迹的地图显示，支持渐变色轨迹线和方向指示
5. **时间筛选**：添加历史位置的时间范围筛选功能，支持多种时间区间选择
6. **视图切换**：支持列表视图和地图视图的切换，提供多样化的数据展示方式
7. **速度显示**：在学生信息卡片、列表视图、历史轨迹中实时显示速度
8. **超速报警**：实现报警按钮闪烁功能，提醒教师有新的超速记录
9. **智能闪烁**：只有新的超速记录才会触发报警按钮闪烁
10. **定位精度优化**：使用高精度定位模式，降低缓存时间和超时时间
11. **定位方式恢复**：从uni.getLocation恢复为plus.geolocation.watchPosition，提高稳定性
12. **容错机制同步**：客户端实现与服务器端相同的100米容错机制
13. **坐标系处理**：APP端使用WGS84坐标系，添加WGS84到GCJ-02的坐标转换，确保与高德地图一致

### 6.3 数据库优化

1. **字段类型调整**：
   - latitude: DECIMAL(10,8)
   - longitude: DECIMAL(11,8)
   确保坐标数据存储精度，避免"Out of range value"错误

2. **速度字段**：
   - speed: DECIMAL(10,2)
   存储学生移动速度，单位：米/秒

3. **SQLAlchemy类型映射说明**：
   - SQLAlchemy的Float类型在默认情况下可能映射为MySQL的FLOAT类型，这会导致精度问题
   - 因此，在models.py中使用Numeric类型替代Float类型，确保映射为MySQL的DECIMAL类型

4. **添加班级字段**：
   - 在students表中添加class_name字段，用于存储学生班级信息
   - 在models.py的Student模型中添加class_name字段
   - 在学生注册API中添加class_name字段的处理

5. **表结构优化**：允许student_id字段为空，提高系统兼容性

### 6.4 常见问题及解决方案

1. **状态显示"未知"**：
   - 确保设备已开启位置权限
   - 检查网络连接是否正常
   - 系统已实现本地边界判断，即使无网络也能显示状态

2. **手动上传失败**：
   - 检查服务器地址配置是否正确
   - 确保服务器已启动并运行
   - 检查数据库连接是否正常

3. **500 Internal Server Error**：
   - 检查数据库表结构是否正确（DECIMAL类型）
   - 查看服务器日志获取详细错误信息

4. **地图功能无法使用**：
   - 原因：uni-app的地图组件需要配置地图模块才能使用
   - 解决方法：
     a. 打开项目的manifest.json文件
     b. 在"App模块配置"中勾选"Maps（地图）"模块
     c. 选择地图服务商（高德地图或腾讯地图）
     d. 填写地图Key（需到对应地图服务商官网申请）
     e. 重新打包应用并安装到手机
   - 地图Key申请：
     - 高德地图：访问 https://lbs.amap.com/
     - 腾讯地图：访问 https://lbs.qq.com/
   - 申请Key时需要填写应用包名，确保与manifest.json中的应用ID一致

5. **定位显示"校外"但实际在校内**：
   - 原因：GPS定位存在误差（通常5-20米）
   - 解决：系统已实现100米容错机制，如果仍然误判，可以：
     a. 检查黄淮学院北区边界坐标是否正确
     b. 调整 `LOCATION_TOLERANCE` 参数（默认100米，可进一步增加）
     c. 确保设备GPS信号良好，在室外开阔地带测试

6. **报警按钮不闪烁**：
   - 检查是否有学生在校外超速（超过80码/35.76米/秒）
   - 检查网络连接是否正常
   - 查看报警页面是否有超速记录

7. **速度始终显示为0**：
   - 原因：缺少运动权限或定位方式不正确
   - 解决：
     a. 确保manifest.json中配置了运动权限
     b. 使用plus.geolocation.watchPosition获取定位数据
     c. 启用高精度定位模式
     d. 在室外开阔地带测试，GPS信号更好
     e. 重新打包APP，确保权限配置生效

8. **manifest.json文件有红线**：
   - 原因：JSON文件不支持注释语法
   - 解决：移除所有注释，确保JSON语法正确
   - 注意：uni-app的manifest.json文件不支持注释，所有注释都需要移除

9. **定位失败**：
   - 原因：APP端plus.geolocation不支持gcj02坐标系
   - 解决：
     a. APP端使用wgs84坐标系
     b. 系统已添加WGS84到GCJ-02的坐标转换
     c. 确保定位参数配置正确

## 7. 技术说明

### 7.1 速度单位转换

- 1 码 = 0.44704 米/秒
- 80 码 = 80 × 0.44704 ≈ 35.76 米/秒
- 系统使用 35.76 米/秒作为超速阈值

### 7.2 报警记录生命周期

- 报警记录会在数据库中保存30天
- 30天后的记录会自动过滤，不再显示
- 教师查看报警后，报警按钮停止闪烁
- 新的超速记录会再次触发闪烁

### 7.3 定位上传频率

- 校内：每3分钟上传一次
- 校外：每1分钟上传一次
- 速度信息随每次上传一起记录

### 7.4 定位精度优化

系统采用了多种优化措施提高定位精度：

1. **高精度定位模式**：
   - 启用 `enableHighAccuracy: true`
   - 降低缓存时间 `maximumAge: 500ms`
   - 降低超时时间 `timeout: 3000ms`

2. **原生定位API**：
   - APP端使用 `plus.geolocation.watchPosition`
   - 比uni.getLocation更稳定，能获取更完整的速度信息

3. **运动权限**：
   - Android端添加 `ACTIVITY_RECOGNITION` 权限
   - iOS端添加 `NSMotionUsageDescription` 配置
   - 确保能够获取运动传感器数据

4. **容错机制**：
   - 100米容错半径，处理GPS定位误差
   - 结合设备定位精度信息，动态调整容错范围
   - 客户端和服务器端保持一致的容错逻辑

### 7.5 坐标系说明

- **服务器端**：使用 WGS84 坐标系（GPS标准坐标系）
- **APP端**：使用 WGS84 坐标系，通过转换函数转换为 GCJ-02
- **H5/小程序端**：直接使用 GCJ-02 坐标系
- **高德地图**：使用 GCJ-02 坐标系（火星坐标系）
- **坐标转换**：APP端已实现 WGS84 到 GCJ-02 的转换，确保与高德地图一致
